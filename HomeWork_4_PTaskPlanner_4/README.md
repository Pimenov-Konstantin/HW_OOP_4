Структура приложения:

1. Controller
    * class InputHander -  ввод и формирование списка задач с консоли, файла
    * class PreemptiveScheduling - инициализация начальных таймингов, формирование очереди
2. Model
    * class GanttRecord - диаграмма планировщика задач.
    * ReadyQueue - операции с очередью (готовность, пустая)
    * class Process - формирование списка задач (Id задачи, время поступления задачи, приоритет, продолжительность выполнеия задачи, )
3. UE
    * class LoadFromFile - загрузка из файлов списка задач
    * class ReadfromFile - вывод на консоль содержимого файла списка задач
    * class WriteToFile - запись в файл свводимых в консоль данных по списку задач

Пояснения к обозначениям:
- GANTT chart - диграмма планирования задач: <br/>
  (время завершения предыдущей задачи)время начала выполнения задачи Id задачи - Id задача <br/> - время завершения Id задачи (время начала другой задачи))


- aTime - arriving time (время поступления задачи )


- bTime - burst time  (продолжительность выполнения задачи)


- Completion Time - время завершения задачи


- Waiting Time - время простоя в очереди задачи
